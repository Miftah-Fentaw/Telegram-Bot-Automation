# Contributing to Telegram-Bot-Automation

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **Telegram-Bot-Automation**. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
- Check the [GitHub Issues](https://github.com/Miftah-Fentaw/Telegram-Bot-Automation/issues) to see if the bug has already been reported.
- If not, create a new issue. Clearly describe the problem, including steps to reproduce and any relevant error logs.

### Suggesting Enhancements
- If you have an idea for a new feature or an improvement, open an issue to discuss it.
- Provide as much detail as possible about the proposed enhancement.

### Pull Requests
- Fork the repository and create your branch from `main`.
- If you've added code that should be tested, add tests.
- Ensure the test suite passes.
- Make sure your code follows the existing style (PEP 8 for Python).
- Write clear, concise commit messages.

## Style Guidelines

- We follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code.
- Use meaningful variable and function names.
- Keep functions small and focused on a single task.

## Community

If you have questions or want to discuss the project, feel free to open an issue or reach out to the maintainers.

Happy coding! 🚀
